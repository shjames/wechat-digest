北京大学与百度联合团队提出 RepoZero，首个面向"从零生成完整代码仓库"的可验证基准。不同于 HumanEval、SWE-bench 等专注单函数或 Bug 修复的评测，RepoZero 要求模型仅凭 API 说明从零构建多文件、跨模块的完整项目，并用真实执行结果严格验证。为防止数据泄露，基准引入跨语言复现任务（Python→JavaScript、C/C++→Rust），强迫模型真正理解算法逻辑而非记忆代码。配套框架 ACE 实现"代码生成→自动测试→错误反馈→修复"的自验证闭环，显著提升通过率。实验表明即便顶级模型在最难任务上通过率仍仅 20%~40%，长上下文遗忘和模块协作错误是主要瓶颈，揭示"真正的软件工程智能"远未实现。
